# 任务计划：标签收藏页一键批量下载与图集绑定方案

## 目标

为 Booru 标签收藏页设计一个新功能方案，使用户可以：

1. 在标签收藏页对某个收藏标签一键发起批量下载；
2. 在每一条标签记录上看到当前下载进度；
3. 看到该标签的上次下载时间；
4. 支持将该标签对应的下载目录与本地图库中的图集建立绑定关系；
5. 在不立即实现代码的前提下，先明确数据模型、数据库改动、服务协调方式、UI 交互和分阶段实现建议。

## 当前阶段

- [completed] Phase 1：现状调研（标签收藏、批量下载、图库模型）
- [in_progress] Phase 2：数据模型与数据库表设计
- [pending] Phase 3：服务协作与事件流设计
- [pending] Phase 4：渲染层交互方案
- [pending] Phase 5：输出正式方案

## 关注点

- 现有标签收藏表是否已足够承载“上次下载时间、绑定图集、下载状态映射”
- 是否需要新增“标签下载配置/状态表”而不是直接改现有收藏标签表
- 现有批量下载任务、会话、记录模型是否能天然承载“标签来源”与“逐条进度”
- 图集绑定应存 galleryId、folderPath，还是两者兼顾
- 页面状态展示应使用实时任务聚合，还是持久化快照字段

## 输出要求

- 方案必须说明需要修改哪些模块
- 方案必须说明建议新增或修改哪些数据表/字段
- 方案必须说明前后端如何协调
- 方案必须说明哪些状态适合持久化，哪些状态适合运行时聚合
- 方案必须给出推荐实现顺序与风险点

## 备注

- 当前任务为方案设计，不做代码实现
- 任何设计结论都必须基于实际代码和现有文档，而不是凭空假设
