# 相册术语统一设计

## 背景与目标

项目当前在中文展示、说明和测试文本中使用同一个旧称。本任务将该旧称统一为“相册”，只调整中文名词语义，不改变任何业务模型、数据结构或接口契约。

旧称的精确字符为 Unicode `U+56FE U+96C6`。替换只匹配这两个连续汉字，不做近义词扩展，也不改写英文 `gallery` / `Gallery`。

## 修改范围

处理 Git 跟踪的项目自有内容，包括：

- Electron 桌面端的用户界面、提示、错误、日志和注释；
- Android 子项目的界面、提示、测试名称、注释和文档；
- TypeScript、TSX、Kotlin 测试中的中文描述、断言文本和夹具文本；
- `README.md`、`doc/`、`docs/` 中的当前文档、历史文档、设计与实施计划；
- 其他受版本控制的项目自有文本文件；
- 文件名中包含旧称的自有文档，并同步修正所有引用。

## 排除范围

- `Boorusama-master-official/**`：作为外部参考源码快照保留原样；
- `.git/**`、`node_modules/**`、`dist/**`、`out/**`、Android `build/**` 等生成物或依赖；
- 数据库表名、字段名、TypeScript/Kotlin 类型名、函数名、变量名、IPC 通道、API 路径与其他英文技术标识；
- 与术语统一无关的格式化、重构或文档改写。

## 实施方式

1. 以 Git 跟踪文件清单作为输入，排除外部参考源码目录。
2. 对文本内容执行精确、机械的中文子串替换，保留原文件编码与换行风格。
3. 将受版本控制且文件名包含旧称的文档改为对应的“相册”名称。
4. 搜索并更新指向这些文档的相对路径、Markdown 链接和文字引用。
5. 对已有未提交文件只修改目标词语，保留用户的其他改动，不进行整文件格式化。

## 风险控制

- 机械替换前后分别统计文件数和匹配数，避免遗漏或误扩大范围。
- 使用 Git diff 检查每一处变更都可追溯到术语替换或文件重命名。
- 对当前已有的未提交改动逐文件核对，确保其无关内容未被覆盖。
- 不删除或清理被忽略的构建产物；后续构建会自然生成使用新测试名称的新产物。

## 验证与完成标准

- Git 跟踪的项目自有文件内容中，旧称精确匹配数为零；
- Git 跟踪的项目自有文件路径中，旧称精确匹配数为零；
- `Boorusama-master-official/**` 的既有匹配保持不变；
- 所有含旧称的自有文档文件均已重命名，仓库内不存在断开的旧路径引用；
- `git diff --check` 通过；
- 运行桌面端测试与构建检查，以及 Android 相关单元测试；若环境本身阻止某项检查，记录完整命令与实际错误，不将其误报为通过；
- 最终 diff 不包含无关格式化或重构。

## 非目标

本任务不重命名代码中的 Gallery 概念，不迁移数据库，不修改网络协议，不调整功能行为，也不重新设计任何界面。
