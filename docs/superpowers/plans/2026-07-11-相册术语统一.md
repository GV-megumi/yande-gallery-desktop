# 相册术语统一 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 将项目自有中文内容中的旧称统一为“相册”，同时保留所有英文技术标识和外部参考源码原样。

**Architecture:** 这是一次原子、无业务行为变化的文本迁移。以 Git 跟踪文件为边界，用精确 Unicode 子串替换处理内容，用 `git mv` 处理两个自有文档路径，再以零匹配审计、完整测试和 Git diff 审查验收。

**Tech Stack:** PowerShell、Git、Node.js、TypeScript/React/Electron、Kotlin/Gradle、Vitest

## Global Constraints

- 只处理 Git 跟踪的项目自有文件；`Boorusama-master-official/**` 必须保持不变。
- 旧称精确字符为 Unicode `U+56FE U+96C6`，新称精确字符为 Unicode `U+76F8 U+518C`。
- 路径说明中的 `<OLD_TERM:U+56FE-U+96C6>` 是非字面占位符；执行命令时必须用上述码点构造 `$old`，受控文档不得重新写入旧称字面。
- 不修改 `gallery` / `Gallery`、数据库表字段、类型、函数、变量、IPC 通道或 API 路径。
- 不清理 `.git/**`、`node_modules/**`、`dist/**`、`out/**`、Android `build/**` 等生成物或依赖。
- 不做无关格式化、重构或文档改写；每个内容变更必须是精确术语替换。
- 原始 checkout `M:\yande\yande-gallery-desktop` 中已有的三个未提交文件必须保持原样、不被暂存或提交；本任务在隔离 worktree 中执行，因此隔离分支可提交这些路径上的纯术语变化。
- 任何 Git commit message 必须使用中文描述。

- [ ] **Step 3: 对受控内容执行精确机械替换**

```powershell
@'
import { execFileSync } from 'node:child_process';
import { readFileSync, writeFileSync } from 'node:fs';

const oldTerm = String.fromCodePoint(0x56fe, 0x96c6);
const newTerm = String.fromCodePoint(0x76f8, 0x518c);
const args = [
  'grep',
  '-Ilz',
  '--',
  oldTerm,
  '--',
  '.',
  ':(exclude)Boorusama-master-official/**',
];

let output = '';
try {
  output = execFileSync('git', args, { encoding: 'utf8' });
} catch (error) {
  if (error?.status !== 1) throw error;
}

const files = output.split('\0').filter(Boolean);
let replacements = 0;

for (const file of files) {
  const normalized = file.replaceAll('\\', '/');
  if (normalized.startsWith('Boorusama-master-official/')) {
    throw new Error(`外部参考源码意外进入替换集合: ${file}`);
  }

  const before = readFileSync(file, 'utf8');
  const count = before.split(oldTerm).length - 1;
  if (count === 0) continue;

  const after = before.replaceAll(oldTerm, newTerm);
  writeFileSync(file, after, 'utf8');

  const written = readFileSync(file, 'utf8');
  if (written !== after) {
    throw new Error(`写入校验失败: ${file}`);
  }
  replacements += count;
}

console.log(JSON.stringify({ filesChanged: files.length, replacements }, null, 2));
'@ | node --input-type=module -
```

Expected: 命令退出 0，`filesChanged` 和 `replacements` 均大于 0。脚本只执行 `before.replaceAll(oldTerm, newTerm)`，因此会保留 BOM、换行序列和目标词以外的全部字符。
