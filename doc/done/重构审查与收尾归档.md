## 分支差异摘要

本分支不是单点文档变更，而是一轮较完整的桌面化、下载链路、IPC/preload、页面闭环和测试补强。与 `master` 对比，主要差异包括：

- 构建链路：新增 `scripts/build-preload.mjs`、`vite.preload.config.ts`，`npm run dev` / `npm run build` 都显式构建 main、preload、renderer 三段。
- preload 分层：主 preload 继续暴露完整 `window.electronAPI`；轻量子窗口新增 `src/preload/subwindow-index.ts`，只暴露 `window` / `booru` / `booruPreferences` / `system`；共享工厂位于 `src/preload/shared/`。
- IPC 与安全边界：通道常量集中到 `src/main/ipc/channels.ts`；主进程对 `app://`、外链打开、窗口导航和 webview 附着增加边界校验。
- 配置与偏好：配置读取改为 renderer-safe DTO；配置变更事件只广播 `ConfigChangedSummary` 摘要；页面偏好进入 `ui.pagePreferences` 分域，支持 App 壳层、收藏标签、黑名单和图库子页状态。
- 下载链路：普通下载使用 `.part` 临时文件、大小 / md5 校验、用户主动暂停 / 取消标志；批量下载增加 `queued`、并发闸门、FIFO 推进、重启恢复闸门和 retry merge 语义。
- 桌面能力：新增托盘、关闭行为、开机自启、启动最小化、桌面通知和通知点击导航。
- 本地图库：图集删除会级联清理元数据 / 缩略图 / 页面偏好，并写入 `gallery_ignored_folders` 防止后续扫描重建；原始图片不物理删除。
- Booru 页面：收藏标签 / 黑名单列表迁移到服务端分页 + 搜索；导入拆成 pickFile + commit；帖子收藏 / 下载 / 服务端喜欢等动作抽出 `useBooruPostActions` 试点复用。
- 导航与窗口：一级菜单、二级菜单、固定页和基础页统一到 `mountedPageIds` 缓存层；`openSecondaryMenu(section, key, tab?, extra?)` 支持在子窗口打开二级菜单并传入上下文。
- Google 页面：Drive / Photos / Gemini 统一采用 webview 嵌入，不再通过 Google API / OAuth 服务层接入。
- 测试：新增主进程生命周期、安全、窗口、preload、批量下载状态机、图库删除、页面渲染和导航等测试。

## 已沉淀到当前文档的规则

### 下载与批量会话

- 任务模板和运行会话必须分开：`bulk_download_tasks` 去重只代表配置已存在，不代表下载还在跑；是否拒绝重启要看同 taskId 是否存在存活会话。
- `pending` / `queued` / `dryRun` / `running` / `paused` 都是会话存活态；其中只有 `dryRun` / `running` 占用并发槽位。
- 超过 `bulkDownload.maxConcurrentSessions` 时写入 `queued`，由 `promoteNextQueued` 在活跃会话离开槽位后推进。
- 用户主动暂停 / 取消的显式标志比错误字符串可靠，错误分支不得把主动中断覆盖为 `failed`。
- 下载文件先写 `.part`，校验通过后替换最终文件；失败或重试只清理临时文件，不删除最终目标。

- `doc/注意事项/下载与批量会话状态机.md`
- `doc/Booru功能实现文档.md`
- `src/main/services/downloadManager.ts`

### IPC / preload / renderer 边界

- Renderer 不直接访问 Node / Electron 主进程能力，统一通过 preload 暴露受控 API。
- `src/main/ipc/channels.ts` 是 IPC 通道常量来源；实际 Renderer 可见 API 还要同时看 `src/preload/index.ts` 和 `src/preload/shared/*.ts`。
- 轻量子窗口不应继承主窗口的高权限 preload；新增轻量页面前必须确认它只依赖 `window` / `booru` / `booruPreferences` / `system`，否则要调整共享工厂或改用二级菜单子窗口。
- 配置事件只下发摘要，渲染端收到后主动拉取 renderer-safe DTO，避免敏感字段通过事件广播扩散。

### 导航、页面偏好和子窗口

- `App.tsx` 使用 `mountedPageIds` 统一承载基础页和固定页缓存，避免同一页面同时存在 base / pinned 双实例。
- 一级菜单切换时先确定目标 section 的当前 subKey；若目标页已固定，走固定页恢复链，否则激活基础页。
- 用户主动返回、关闭、删除这类明确动作需要同步落盘或显式清空偏好；不要只依赖防抖 effect。
- 子窗口通过 URL query / prop 传上下文，不能借 `pagePreferences` 传临时状态；必要时禁用子窗口对主窗口偏好的写回。

### 本地图库删除与扫描恢复

- `images` 表没有 `thumbnailPath`，普通缩略图路径由缩略图服务按原图路径推导；写 SELECT 前必须核对 `database.ts`。
- 删除图集只清理数据库元数据、缩略图缓存、Booru 本地关联和页面偏好，不删除原图文件。
- 删除图集会写入 `gallery_ignored_folders`，扫描 / 同步链路命中后整棵子树跳过，避免删完又重建。
- 非递归图集只处理直接子文件，不能误删子目录里可能属于其他图集的图片。

### 桌面行为、通知和安全

- 通知走 `notificationService` 统一判断：全局 `notifications.enabled`、按状态 `byStatus`、任务级 `notifications` 三层同时为真才弹批量下载通知。
- `desktop.closeAction` 的 `hide-to-tray` / `quit` / `ask` 语义要和托盘生命周期、退出清理链路一致。
- `app://` 只映射图库目录、下载目录和数据目录等受控根；外链只允许 https 且禁止本机 / 内网目标；webview 只允许白名单 HTTPS 主机。
- 轻量子窗口和 webview 都要遵守最小暴露面，不要为了方便把主 preload 或 Node 能力带过去。

### Antd 表单与 UI 约定

- `Form.Item name` 的直接子组件必须是真正受控输入；`Space.Compact` / `div` 只能放在外层布局。
- Modal 不要同时保留默认 X 和自绘关闭按钮；创建 / 编辑共用弹窗应使用 `destroyOnHidden` 或稳定 key，避免表单串味。
- `InputNumber max` 是静默 clamp，产品没有硬上限时不要随手写 max；需要约束时优先用规则校验给用户明确反馈。
- 图标按钮、右键菜单、删除确认、返回 / 关闭语义应统一，不要在每个页面自定义一套。
