# 按钮与交互优化方案

## 一、现状总览

### 1.1 按钮出现的三个层级

| 层级 | 位置 | 当前按钮 | 说明 |
|------|------|---------|------|
| **卡片 Overlay** | BooruImageCard 右上角 | 预览、收藏、喜欢（条件）、下载 | 毛玻璃圆形按钮，始终可见 |
| **右键菜单** | BooruImageCard + 标签 | 预览、收藏、喜欢、下载、复制链接、浏览器打开 | ContextMenu 组件 |
| **详情页工具栏** | BooruPostDetails/Toolbar | 收藏、喜欢、点赞、踩、下载、幻灯片、分享 | Ant Design Button |

### 1.2 各页面"喜欢"功能接入情况

| 页面 | 卡片喜欢按钮 | 详情页喜欢 | 状态 |
|------|------------|-----------|------|
| BooruPage（主浏览） | ✅ 已接入 | ✅ | 正常 |
| BooruFavoritesPage（收藏） | ❌ 未传递 | ❌ | **缺失** |
| BooruTagSearchPage（标签搜索） | ❌ 未传递 | ❌ | **缺失** |
| BooruServerFavoritesPage（喜欢列表） | ❌ 未传递 | ❌ | **缺失**（最不合理） |
| BooruPopularPage（热门） | ❌ 未传递 | ❌ | **缺失** |

## 二、发现的问题

### 2.1 功能缺失

**P0 - 喜欢功能只在主浏览页可用**

"喜欢"按钮（Heart）目前只有 `BooruPage` 传递了 `onToggleServerFavorite`，其他所有使用 `BooruGridLayout` 或 `BooruImageCard` 的页面都没有接入。最不合理的是"我的喜欢"列表页（BooruServerFavoritesPage）本身都没有取消喜欢的快捷操作。

**P1 - 详情页工具栏功能未回传卡片列表**

详情页 Toolbar 的喜欢/收藏操作结果没有同步回卡片列表的状态（部分页面），导致详情页操作后返回列表时状态可能不一致。

### 2.2 交互体验问题

**P0 - Overlay 按钮始终可见，遮挡图片**

当前 4 个（甚至 5 个）Overlay 按钮始终显示在图片右上角，占用大量空间，尤其在小 gridSize 下严重遮挡图片内容。

建议：改为 hover 时才显示，带渐入/渐出动画。

**P1 - Overlay 按钮过小，难以点击**

按钮尺寸 30x30px，在密集网格中不易精确点击，且缺少 active（按下）状态反馈。

**P2 - 收藏和喜欢按钮激活色相同**

收藏（Book）和喜欢（Heart）激活后都是 `rgba(255, 45, 85, 0.85)` 粉红色，用户无法通过颜色快速区分两个功能的状态。

### 2.3 按钮布局问题

**P1 - 卡片 Overlay 按钮排列方向不合理**

4 个按钮水平排列在右上角，当卡片宽度较小时容易溢出或过于拥挤。

建议：改为垂直排列（竖排），或分组放置（左上角 + 右上角）。

**P2 - 右键菜单与 Overlay 按钮功能完全重复**

预览、收藏、喜欢、下载同时出现在 Overlay 和右键菜单中。右键菜单应该提供 Overlay 没有的高级功能，避免冗余。

### 2.4 样式不一致

**P2 - Overlay 按钮与详情页按钮风格割裂**

- 卡片 Overlay：毛玻璃圆形按钮，无文字
- 详情页 Toolbar：Ant Design 标准按钮，有图标 + 文字

两种风格差异过大，缺乏统一的设计语言。

## 三、优化方案

### 3.1 Overlay 按钮改为 Hover 显示

**目标**：减少视觉噪音，让用户聚焦图片内容。

**实现**：
- 默认隐藏所有 Overlay 按钮（opacity: 0）
- 鼠标悬停卡片时，按钮渐入显示（opacity: 1，transition: 0.2s）
- 已激活的按钮（已收藏/已喜欢）始终显示，不受 hover 影响
- 添加卡片底部渐变遮罩，提升按钮可读性

```
默认状态：           Hover 状态：
┌─────────────┐     ┌─────────────┐
│             │     │        ◉◉◉◉│  ← 按钮渐入
│   图片内容   │     │   图片内容   │
│             │     │             │
│             │     │▒▒▒▒▒▒▒▒▒▒▒▒│  ← 底部渐变遮罩
│  站点 评分   │     │  站点 评分   │
│  标签...     │     │  标签...     │
└─────────────┘     └─────────────┘

已收藏/已喜欢时：
┌─────────────┐
│        ♥ ◆  │  ← 激活按钮始终可见
│   图片内容   │
│             │
│  站点 评分   │
│  标签...     │
└─────────────┘
```

### 3.2 Overlay 按钮布局优化

**方案：右上角竖排**

将按钮从水平排列改为垂直排列，减少水平空间占用：

```
┌──────────────────┐
│              [👁] │  预览
│              [♥] │  喜欢
│              [◆] │  收藏
│              [↓] │  下载
│                  │
│    图片内容       │
│                  │
└──────────────────┘
```

**按钮顺序调整**：
1. 预览（Eye）— 最常用，放最上面
2. 喜欢（Heart）— 服务端操作
3. 收藏（Book）— 本地操作
4. 下载（Download）— 放最下面

**尺寸优化**：
- 按钮尺寸从 30px 增大到 32px
- 间距从 6px 调整为 6px（垂直间距）
- 添加 active（按下）状态：`transform: scale(0.9)`

### 3.3 区分收藏和喜欢的视觉样式

| 按钮 | 未激活 | 激活色 | 图标 |
|------|--------|-------|------|
| 喜欢（Heart） | 半透明黑 | `rgba(255, 45, 85, 0.85)` 粉红 | HeartOutlined → HeartFilled |
| 收藏（Book） | 半透明黑 | `rgba(255, 149, 0, 0.85)` 橙色 | BookOutlined → BookFilled |
| 预览（Eye） | 半透明黑 | 无激活态 | EyeOutlined |
| 下载（Download） | 半透明黑 | 下载中显示加载动画 | DownloadOutlined |

关键改动：收藏按钮激活色改为**橙色**，与喜欢的粉红色区分。

### 3.4 统一所有页面的喜欢功能

需要修改的页面：

1. **BooruFavoritesPage** — 添加 `onToggleServerFavorite` 和 `serverFavorites` 状态
2. **BooruTagSearchPage** — 传递 `onToggleServerFavorite` 到 BooruGridLayout
3. **BooruServerFavoritesPage** — 实现喜欢状态管理，允许在列表中取消喜欢
4. **BooruPopularPage** — 传递 `onToggleServerFavorite` 到 BooruImageCard

每个页面需要：
- 获取 `activeSite` 的登录状态
- 维护 `serverFavorites: Set<number>` 状态
- 实现 `handleToggleServerFavorite` 回调
- 传递给 BooruGridLayout / BooruImageCard

### 3.5 右键菜单优化

**原则**：右键菜单提供 Overlay 按钮没有的补充功能。

```
当前右键菜单：              优化后：
├─ 预览          (重复)     ├─ 预览
├─ 收藏/取消收藏  (重复)     ├─ 收藏/取消收藏
├─ 喜欢/取消喜欢  (重复)     ├─ 喜欢/取消喜欢
├─ 下载          (重复)     ├─ 下载
├─ ────────                ├─ ────────
├─ 复制图片链接              ├─ 复制图片链接
└─ 在浏览器中打开            ├─ 复制预览图链接     (新增)
                           ├─ 在浏览器中打开
                           ├─ ────────
                           ├─ 以图搜图           (新增，可选)
                           └─ 查看图片信息        (新增，显示尺寸/格式/大小)
```

保留基础操作（预览、收藏、喜欢、下载）是合理的，因为右键菜单是触屏和键盘用户的备选操作方式。新增的项提供了 Overlay 无法承载的功能。

### 3.6 添加按钮交互反馈

**按下反馈（active 状态）**：
```css
button:active {
  transform: scale(0.9);
  transition: transform 0.1s;
}
```

**操作成功反馈**：
- 收藏成功：按钮短暂放大弹跳动画
- 喜欢成功：心形图标短暂放大 + 粒子效果（可选）
- 下载成功：图标切换为勾号，1 秒后恢复

**操作失败反馈**：
- 按钮短暂抖动动画
- 显示 message.error 提示

## 四、详情页交互问题

### 4.1 上一张/下一张导航位置不合理（P0）

**当前**：关闭按钮、上一张/下一张全部挤在左上角工具栏

```
当前布局：
┌─[✕ 关闭]─[< 1/15 >]──────────────────────[ID: 1256178]─┐
│                                           │  标题        │
│                                           │  艺术家      │
│              图 片 区 域                    │  工具栏按钮  │
│                                           │  标签        │
│                                           │  文件详情    │
└───────────────────────────────────────────┴─────────────┘
```

**问题**：
- 上一张/下一张远离图片，操作路径长
- 关闭和导航混在一起，容易误触
- 不符合图片浏览器的通用交互习惯（如 Windows 照片、macOS 预览）

**优化方案**：导航按钮移到图片区域左右两侧，半透明悬浮

```
优化后布局：
┌─[✕ 关闭]───────────────[1/15]────────────[ID: 1256178]─┐
│                                           │  标题        │
│  [<]        图 片 区 域              [>]   │  工具栏按钮  │
│  半透明       hover 时显示           半透明 │  标签        │
│                                           │  文件详情    │
└───────────────────────────────────────────┴─────────────┘
```

**导航按钮样式**：
- 尺寸：40x80px 矩形，圆角
- 背景：`rgba(0, 0, 0, 0.2)`，hover 时 `rgba(0, 0, 0, 0.5)`
- 默认半透明，hover 图片区域时完全显示
- 保留键盘快捷键 `←` `→` `Esc`

### 4.2 详情页工具栏按钮分组不合理（P1）

**当前**：收藏、喜欢、点赞、踩、下载、幻灯片、分享 — 7 个按钮平铺在一行

**问题**：
- 按钮过多，在窄屏下换行导致布局不稳定
- 主要操作（收藏、下载）和次要操作（幻灯片、分享）没有视觉区分
- 点赞和踩两个按钮可以合并为评分组

**优化方案**：按钮分为两行/两组

```
主操作行：  [收藏]  [喜欢]  [👍] [👎]
次操作行：  [下载]  [幻灯片]  [分享]
```

### 4.3 详情页右侧面板标签缺少右键操作（P1）

**当前**：TagsSection 已有右键菜单（复制、搜索、收藏、黑名单）

**问题**：截图中"未知艺术家"等红色标签（艺术家标签）虽然有右键菜单，但：
- 用户不容易发现右键菜单的存在
- 建议在标签上添加 hover 时显示操作提示（如 tooltip："右键查看更多操作"）
- 或者在标签旁添加小型操作图标（hover 时显示）

## 五、其他页面交互问题

### 5.1 热门页筛选栏布局抖动（P2）

**问题**：右侧选择器数量根据 period 值动态变化（Select / DatePicker 切换），导致布局抖动。

**优化**：固定筛选栏高度和宽度，预留空间。

### 5.2 Pool 页面缺少返回导航（P2）

**问题**：进入 Pool 详情后，缺少清晰的返回按钮或面包屑导航。

**优化**：添加返回按钮和面包屑路径。

### 5.3 下载页表格行操作反馈不足（P2）

**问题**：
- 删除按钮没有 loading 状态
- "查看原图"没有加载提示
- 用户不知道可以右键点击表格行

**优化**：
- 添加 loading 状态
- 右键菜单添加视觉提示

### 5.4 批量下载页操作按钮缺少 loading（P2）

**问题**：暂停/恢复/取消等异步操作没有 loading 状态反馈。

### 5.5 异步操作统一 loading 反馈（P2）

**问题**：全项目中的按钮操作缺乏统一的 loading 和 error 反馈模式。

**已有 loading**：CommentSection、Toolbar 投票/收藏
**缺失 loading**：下载页删除操作、批量下载暂停/恢复、设置页保存
