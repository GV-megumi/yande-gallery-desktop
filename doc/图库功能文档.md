# 图库功能文档

## 文档定位

本文件描述当前本地图库子系统的结构与设计重点，去掉早期搭建阶段的伪代码和过时路径示例。

## 当前能力

- 最近图片浏览
- 最近图片缓存恢复增量刷新
- 全部图片分页浏览
- 相册 / 图库管理
- 相册文件夹同步
- 图库 CRUD（设置页直接操作数据库：新增、彻底删除、停止监视）
- 相册删除与忽略名单
- 无效图片管理
- 缩略图缓存
- 标签搜索与批量关联
- 图片预览、缩放、拖拽、键盘导航
- 图片删除（含数据库记录、磁盘文件、缩略图）
- 相册卡片右键"用单独窗口打开"

## 主进程模块

- `imageService.ts`：图片导入、查询、标签处理
- `galleryService.ts`：图库 CRUD、封面、目录扫描
- `galleryRootRegistry.ts`：进程内同步缓存图库根路径（启动时从 DB 装载，创建/删除时增量维护；供 `app://` 白名单使用）
- `thumbnailService.ts`：缩略图生成与缓存
- `database.ts`：表结构与索引初始化

## 渲染层页面与组件

- 页面：`src/renderer/pages/GalleryPage.tsx`
- 核心组件：`ImageGrid.tsx`、`ImageListWrapper.tsx`、`ImageSearchBar.tsx`、`LazyLoadFooter.tsx`、`GalleryCoverImage.tsx`

## 设计重点

### 1. 图库与图片分层

- 图库本身是目录级对象，持久化在 `galleries` 表。
- 图片元数据持久化在 `images` 表。
- 图库封面通过 `coverImageId` 指向图片。

### 2. 懒加载优先

- 图库列表优先展示目录元数据。
- 真实导入和扫描尽量按需触发。
- 缩略图优先于原图，用于控制初始渲染成本。

### 3. 相册文件夹同步

- `syncGalleryFolder(id)` 对已有相册执行重新扫描，自动导入目录中新增的图片文件并更新图片统计信息。
- 与初次 `scanAndImportFolder` 的区别：同步是增量操作，只导入尚未入库的文件，不影响已有记录。
- 返回 `{ imported, skipped, imageCount, lastScannedAt }`，前端据此刷新相册卡片数据。
- 相册创建、编辑、删除、统计和封面变化会广播 `gallery:galleries-changed`；已缓存的相册页据此刷新列表和当前打开相册信息。

### 4. 无效图片管理

- 当图片文件在磁盘上不存在或损坏时，可通过 `reportInvalidImage(imageId)` 标记为无效。
- 提供 `getInvalidImages` / `getInvalidImageCount` / `deleteInvalidImage` / `clearInvalidImages` 一套管理接口。
- 无效图片上报、删除或清空会广播 `gallery:invalid-images-changed`，并在影响相册统计或封面时同步广播 `gallery:galleries-changed`。

### 5. 图库管理：彻底删除 vs 停止监视

图库定义持久化于数据库 `galleries` 表，设置页「图库文件夹」直接对其 CRUD；旧 `config.galleries.folders` 配置已移除，首次启动时**若数据库图库为空且**检测到旧残留，会一次性迁移进数据库（DB 已有图库则跳过）。

两种终止监视方式：

- **彻底删除**（`deleteGallery(id)`）：级联清理数据库记录和缩略图缓存，原始磁盘图片保留不删。`deleteGallery` 会根据目标相册的 `recursive` 字段决定清理范围：
  - `recursive=1`：整棵子树（`folderPath LIKE prefix%`）的 `images` / 缩略图 / `booru_posts.localImageId` / `invalid_images` / 页面偏好中残留的 `selectedGalleryId` 一并清理。
  - `recursive=0`：只清理相册目录下的直接子文件，保留深层子目录的图片记录。
  - 删除后目录会被写入 `gallery_ignored_folders`，扫描 / 同步链路会整棵子树跳过，避免"删了又被扫描重建"。
- **停止监视**（`updateGallery(id, { isWatching: false })`）：保留所有数据库记录，仅关闭自动同步；目录不写入忽略名单，随时可重新启用。

忽略名单相关：

- 忽略名单可通过 `listIgnoredFolders / addIgnoredFolder / updateIgnoredFolder / removeIgnoredFolder` 维护。
- 忽略名单新增、编辑或删除会广播 `gallery:ignored-folders-changed`。

相关约束见 `doc/注意事项/数据库与删除级联规范.md`。

### 6. 最近图片缓存与增量刷新

最近图片页有两套不同加载语义：

- **完整进入**：用户首次进入最近图片页，或从图库二级菜单切到其它页导致最近页缓存释放后再次进入，继续走 `getRecentImages(2000)` 的完整加载逻辑。
- **缓存恢复**：用户只是切换一级菜单（例如最近图片 → Booru → 最近图片）或通过固定项恢复，最近图片页仍在 App 缓存层中，此时不得重新加载旧列表；只允许查询并追加新增图片。

缓存恢复时的新增图片处理规则：

- 用当前顶部图片的 `updatedAt + id` 作为游标，通过 `getRecentImagesAfter(updatedAt, id, limit)` 查询更新的图片。
- 主进程扫描或同步导入新图片后会广播 `gallery:images-imported`；最近图片页只在当前处于 recent 且未挂起时消费该事件，并复用同一套增量加载逻辑。
- 图片新增、删除、标签更新、无效化或批量导入会广播 `gallery:images-changed`；`GalleryPage`、`ImageGrid` 等缓存消费者通过 `useGalleryDomainEvents` 按影响范围刷新或剔除本地数组。
- 新增图片放在页面顶部的独立“新增图片”区域，不合并进旧日期组，避免旧瀑布流重新排版。
- 新增区域按最多 20 张拆成分块；只有最顶部未满 20 张的活动块可以继续变化，满 20 张后冻结。
- 原始最近图片仍按 200 张一批冻结；继续下拉时追加 201-400、401-600 等独立历史块。
- 用户已滚动到较深位置时，不立即把新增图片插到顶部，先显示“新增 N 张，点击查看”提示，点击后再插入并回到顶部，避免滚动位置突然跳动。

这个设计的核心不变量是：**缓存恢复只增加页面长度，不重排已冻结的旧块**。

### 8. Gallery 领域事件

图库相关主进程写入会发布统一 `RendererAppEvent`，当前事件包括：

- `gallery:images-imported`
- `gallery:images-changed`
- `gallery:galleries-changed`
- `gallery:invalid-images-changed`
- `gallery:ignored-folders-changed`
- `thumbnail:generated`

Renderer 页面和组件优先通过 `useGalleryDomainEvents` 消费这些事件；具体 payload 以 `src/shared/appEvents.ts` 和 `doc/Renderer API 文档.md` 为准。

### 9. 搜索与浏览模式分离

- 最近图片、全部图片、相册浏览的加载策略并不完全相同。
- 页面层通过不同状态和组件组合处理不同模式，避免互相污染。
