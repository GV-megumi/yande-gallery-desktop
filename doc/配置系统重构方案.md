# 配置系统重构方案

> 状态：**已完成**

## 一、重构前的问题

### 路径解析混乱

| 模块 | 旧做法 | 问题 |
|------|--------|------|
| `config.ts` | `__dirname + '../../../config.yaml'` | 依赖编译输出目录层级 |
| `config.ts` `getAbsolutePath()` | `__dirname + '../../..'` + 相对路径 | 同上 |
| `database.ts` | `__dirname + '../../../data'` | 硬编码，不读 config |
| `imageCacheService.ts` | `process.cwd() + 'data/cache'` | 硬编码，不读 config |
| `thumbnailService.ts` | config → `getAbsolutePath()` | 基准目录是 `__dirname` |
| `downloadManager.ts` | `getDownloadsPath()` | 通过 config，正确 |

- 打包后 `__dirname` 在 asar 内不可写，`process.cwd()` 不可控
- 无法自定义数据存储位置
- 没有 `.env` 支持

## 二、重构后的目录结构

```
CONFIG_DIR/                          默认 ~/.yandegallery（由 .env 的 CONFIG_DIR 指定）
  config.yaml                        应用配置文件
  gallery.db                         SQLite 数据库（resolveConfigPath）
  data/                              运行数据目录（由 config.yaml 的 dataPath 指定）
    thumbnails/                      缩略图缓存（resolveDataPath）
    cache/                           Booru 图片缓存（getCachePath）
    app.log                          应用日志（resolveDataPath）
```

### 路径解析链

```
.env CONFIG_DIR
  → configDir（默认 ~/.yandegallery）
    ├── config.yaml
    ├── gallery.db                   ← resolveConfigPath(database.path)
    └── dataPath: data               ← config.yaml 中指定
          → dataDir（默认 configDir/data）
              ├── thumbnails/        ← resolveDataPath(thumbnails.cachePath)
              ├── cache/             ← getCachePath()
              └── app.log            ← resolveDataPath(logging.filePath)
```

### 两类路径解析函数

| 函数 | 基准目录 | 用途 |
|------|---------|------|
| `resolveConfigPath(path)` | `configDir` | 数据库等与 config.yaml 同级的文件 |
| `resolveDataPath(path)` | `dataDir` | 缩略图、缓存、日志等运行数据 |

所有路径字段均支持绝对路径，绝对路径时忽略基准目录直接返回。

## 三、已完成的修改

### 3.1 新增文件

| 文件 | 说明 |
|------|------|
| `.env.example` | 环境变量模板，说明 CONFIG_DIR |
| `.env` | 开发环境用 `CONFIG_DIR=.`（已在 .gitignore 中忽略） |
| `config.example.yaml` | 更新为新格式，添加完整注释 |

### 3.2 config.yaml 格式变更

```yaml
# 新增顶层字段
dataPath: data                    # 运行数据目录，相对于 CONFIG_DIR

database:
  path: gallery.db                # 相对于 CONFIG_DIR（旧: data/gallery.db）

thumbnails:
  cachePath: thumbnails           # 相对于 dataPath（旧: data/thumbnails）

logging:
  filePath: app.log               # 相对于 dataPath（旧: data/app.log）

# downloads.path / galleries.folders[].path 不变，建议使用绝对路径
```

### 3.3 重构 `config.ts`

**删除**：
- `getAbsolutePath()` — 基于 `__dirname` 的旧路径解析
- `fileURLToPath` / `__dirname` / `__filename` — 不再依赖编译输出目录

**新增**：
- `initPaths()` — 加载 .env → 确定 configDir → 确保目录存在
- `getConfigDir()` / `getDataDir()` — 获取两个基准目录
- `resolveConfigPath()` — 相对于 configDir 解析路径
- `resolveDataPath()` — 相对于 dataDir 解析路径
- `getCachePath()` — `dataDir/cache`
- `getLogFilePath()` — `resolveDataPath(logging.filePath)`
- `ensureDataDirectories()` — 确保所有子目录存在
- `migrateOldPaths()` — 自动去掉旧格式的 `data/` 前缀
- `migrateFromAppDirectory()` — 旧项目目录自动迁移到 configDir

**修改**：
- `loadConfig()` — 从 `configDir/config.yaml` 加载
- `saveConfig()` — 同上
- `getDatabasePath()` — 改用 `resolveConfigPath()`
- `getThumbnailsPath()` — 改用 `resolveDataPath()`
- `getDownloadsPath()` — 绝对路径直接返回，相对路径基于 dataDir

### 3.4 重构 `database.ts`

- 删除硬编码 `DB_DIR` / `DB_PATH` 常量和 `initDbDirectory()` 函数
- `getDatabase()` 改为调用 `getDatabasePath()` 获取路径
- `isDatabaseInitialized()` 同样使用 `getDatabasePath()`

### 3.5 重构 `imageCacheService.ts`

- 5 处 `path.join(process.cwd(), 'data', 'cache')` 全部替换为 `getCachePath()`
- 本地函数 `getCachePath()` 重命名为 `getCacheFilePath()` 避免与 config 导出冲突

### 3.6 修改 `init.ts` 启动流程

```
initializeApp()
  → initPaths()                   // 0. 读 .env → 确定 configDir
  → loadConfig()                  // 1. 加载 config.yaml → 确定 dataDir
  → ensureDataDirectories()       // 1.5. 创建所有子目录
  → initDatabase()                // 2. 初始化数据库
  → initGalleriesFromConfig()     // 3. 初始化图库
  → resumeDownloadsInBackground() // 4. 恢复下载
```

### 3.7 数据文件迁移

- `data/gallery.db` → `./gallery.db`（移到项目根目录 = CONFIG_DIR）
- `data/thumbnails/` 和 `data/cache/` 保持不动（已在 dataPath 下）

### 3.8 依赖变更

- 新增 `dotenv` 依赖

## 四、迁移兼容策略

### 自动迁移（首次启动检测）

1. **旧项目迁移**：如果 configDir 下没有 config.yaml，但程序目录下有：
   - 复制 `config.yaml` → `configDir/config.yaml`
   - 复制 `data/gallery.db` → `configDir/gallery.db`
   - 复制 `data/` → `configDir/data/`

2. **旧路径格式兼容**：`migrateOldPaths()` 自动处理
   - `data/gallery.db` → `gallery.db`
   - `data/thumbnails` → `thumbnails`
   - `data/app.log` → `app.log`

### 开发模式

`.env` 中设置 `CONFIG_DIR=.` 即可使用项目目录作为 configDir，行为与重构前一致。
