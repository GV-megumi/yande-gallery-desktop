# 数据库结构文档

## 说明

当前数据库 schema 以 `src/main/services/database.ts` 为准；本文件只做结构总览和数据域说明，不再重复完整初始化 SQL。

## 数据库位置

- 数据库路径由 `src/main/services/config.ts` 解析。
- 默认情况下，`database.path` 基于配置根目录解析，默认文件名是 `gallery.db`。

## 表结构分组

### 1. 本地图库

- `images`：本地图片元数据
- `tags`：本地标签
- `image_tags`：本地图片和标签的关联
- `galleries`：图库 / 图集元数据
- `yande_images`：早期 Yande 记录表，仍保留兼容

### 2. Booru 核心

- `booru_sites`
- `booru_posts`
- `booru_tags`
- `booru_post_tags`
- `booru_favorites`
- `booru_download_queue`
- `booru_search_history`

### 3. 批量下载

- `bulk_download_tasks`
- `bulk_download_sessions`
- `bulk_download_records`
- `bulk_download_session_stats`

### 4. 收藏标签与黑名单

- `booru_favorite_tags`
- `booru_favorite_tag_labels`
- `booru_blacklisted_tags`

### 5. 收藏分组与保存的搜索

- `booru_favorite_groups`
- `booru_saved_searches`

### 6. 其他扩展表

数据库初始化里还会补充一些扩展字段和索引，用于缓存、进度统计和兼容迁移。

## 关键关系

### 本地图库关系

- `image_tags.imageId -> images.id`
- `image_tags.tagId -> tags.id`
- `galleries.coverImageId -> images.id`

### Booru 关系

- `booru_posts.siteId -> booru_sites.id`
- `booru_posts.localImageId -> images.id`
- `booru_tags.siteId -> booru_sites.id`
- `booru_post_tags.postId -> booru_posts.id`
- `booru_post_tags.tagId -> booru_tags.id`
- `booru_favorites.postId -> booru_posts.id`
- `booru_download_queue.postId -> booru_posts.id`

### 批量下载关系

- `bulk_download_tasks.siteId -> booru_sites.id`
- `bulk_download_sessions.taskId -> bulk_download_tasks.id`
- `bulk_download_records.sessionId -> bulk_download_sessions.id`
- `bulk_download_session_stats.sessionId -> bulk_download_sessions.id`

## 数据建模要点

### 本地图库与 Booru 分离建模

- 本地图库依旧围绕 `images` 建模。
- Booru 数据围绕 `booru_posts` 建模。
- 通过 `localImageId` 在需要时把远端帖子和本地图像关联起来。

### 站点能力通过 `booru_sites.type` 区分

- 当前支持 `moebooru`、`danbooru`、`gelbooru`。
- 不同站点功能不对齐，因此 schema 采用“统一主表 + 能力差异在 service 层处理”的做法。

### 索引策略

- 查询频繁的状态、时间、站点、标签、下载记录都有独立索引。
- 具体索引名称与新增字段请直接查看 `src/main/services/database.ts`。
