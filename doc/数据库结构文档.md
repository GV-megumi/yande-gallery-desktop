# 数据库结构文档

## 说明

当前数据库 schema 以 `src/main/services/database.ts` 为准。本文件升级为“分表字段表格”版本，方便直接查字段、约束、关系和兼容迁移点。

## 数据库位置与初始化

- 数据库路径由 `src/main/services/config.ts` 的 `getDatabasePath()` 决定
- 默认文件名是 `gallery.db`，默认与 `config.yaml` 同级
- 初始化入口：`src/main/services/init.ts` -> `initializeApp()` -> `initDatabase()`
- 连接 PRAGMA：`journal_mode=WAL`、`synchronous=NORMAL`、`cache_size=-8000`、`foreign_keys=ON`、`temp_store=MEMORY`

| `filepath` | `TEXT NOT NULL UNIQUE` | 完整路径，唯一 |
| `fileSize` | `INTEGER NOT NULL` | 文件大小 |

| `height` | `INTEGER NOT NULL` | 图片高度 |
| `format` | `TEXT NOT NULL` | 图片格式 |
| `createdAt` | `TEXT NOT NULL` | 创建时间 |
| `updatedAt` | `TEXT NOT NULL` | 更新时间 |

#### `tags`

本地标签表。

| 字段 | 类型 / 约束 | 说明 |
| --- | --- | --- |
| `id` | `INTEGER PRIMARY KEY AUTOINCREMENT` | 主键 |
| `name` | `TEXT NOT NULL UNIQUE` | 标签名 |
| `category` | `TEXT` | 标签分类 |
| `createdAt` | `TEXT NOT NULL` | 创建时间 |

#### `image_tags`

本地图片和标签的多对多关联表。

| 字段 | 类型 / 约束 | 说明 |
| --- | --- | --- |
| `imageId` | `INTEGER NOT NULL` | 关联 `images.id` |
| `tagId` | `INTEGER NOT NULL` | 关联 `tags.id` |

| 字段 | 类型 / 约束 | 说明 |
| --- | --- | --- |
| `id` | `INTEGER PRIMARY KEY AUTOINCREMENT` | 主键 |
| `folderPath` | `TEXT NOT NULL UNIQUE` | 图库目录完整路径 |
| `name` | `TEXT NOT NULL` | 图库名称 |

#### `gallery_ignored_folders`

被显式加入忽略名单的图库目录路径。`deleteGallery` 会把被删图集的目录写入这里，扫描 / 同步链路读取它来跳过整棵子树，避免"删完又被自动重建"。

| 字段 | 类型 / 约束 | 说明 |
| --- | --- | --- |
| `id` | `INTEGER PRIMARY KEY AUTOINCREMENT` | 主键 |
| `folderPath` | `TEXT NOT NULL UNIQUE` | 归一化后的目录路径 |
| `note` | `TEXT` | 忽略原因，可选 |
| `createdAt` | `TEXT NOT NULL` | 加入时间 |
| `updatedAt` | `TEXT NOT NULL` | 更新时间 |

补充说明：

- 索引：`idx_gallery_ignored_folders_folderPath`
- 无外键；直接用 `folderPath` 与 `galleries` / `images.filepath` 做前缀匹配

## 初始化时的兼容迁移与动态加列

### 动态加列

初始化过程中会按需补以下列：

- `bulk_download_records.progress`
- `bulk_download_records.downloadedBytes`
- `bulk_download_records.totalBytes`
- `bulk_download_sessions.originType`
- `bulk_download_sessions.originId`
- `booru_favorites.groupId`
- `booru_posts.isLiked`
- `booru_tags.updatedAt`
- `booru_favorite_tag_download_bindings.autoCreateGallery`
- `booru_favorite_tag_download_bindings.autoSyncGalleryAfterDownload`

这意味着即使旧数据库缺这些列，应用启动后也会自动补齐。

### `booru_favorites.postId` 迁移

- 旧版本把 API 的 `postId` 直接写进 `booru_favorites.postId`
- 当前逻辑要求它引用 `booru_posts.id`
- 初始化时会尝试把旧值映射到正确的内部主键，并删除无法映射的孤儿行

### 默认站点初始化

数据库初始化时会 `INSERT OR IGNORE` 三个默认站点：

- `Yande.re`
- `Konachan.com`
- `Konachan.net`

对一个全新的配置目录来说，数据库初始化完成后通常就会先拥有这三个默认 Moebooru 站点记录。

## 索引策略

当前索引分三类：

1. 基础索引：文件路径、创建时间、更新时间、标签名、站点、状态等单列索引
2. 复合索引：如 `booru_posts(siteId, createdAt DESC)`、`booru_posts(siteId, downloaded)`、`booru_posts(siteId, rating)`、`bulk_download_records(sessionId, status)`
3. 兼容与性能补充索引：如 `idx_booru_favorites_postId`、`idx_booru_posts_localImageId`、`idx_image_tags_tagId`

这里的索引说明是主题级摘要，不是完整索引清单。如果需要精确索引名、部分索引条件或默认值，请直接看 `src/main/services/database.ts`。

## 关键唯一约束速查

- `images.filepath UNIQUE`
- `tags.name UNIQUE`
- `galleries.folderPath UNIQUE`
- `yande_images.yandeId UNIQUE`
- `booru_sites.url UNIQUE`
- `booru_posts UNIQUE(siteId, postId)`
- `booru_tags UNIQUE(siteId, name)`
- `booru_favorites UNIQUE(postId)`
- `booru_favorite_tags UNIQUE(siteId, tagName)`
- `booru_blacklisted_tags UNIQUE(siteId, tagName)`
- `booru_saved_searches` 当前没有名称唯一约束

补充说明：对 `booru_favorite_tags`、`booru_blacklisted_tags` 这类 `siteId` 可空的表，SQLite 中 `UNIQUE(siteId, tagName)` 对 `siteId = NULL` 的全局行并不提供严格的“只能有一条”保证；如果需要强约束，需要额外业务逻辑配合。

## 数据建模要点

### 本地图库与 Booru 分离建模

- 本地图库围绕 `images` 建模
- Booru 数据围绕 `booru_posts` 建模
- 两者通过 `localImageId` 在需要时建立关联

### 站点能力通过 `booru_sites.type` 区分

- 当前支持 `moebooru`、`danbooru`、`gelbooru`
- schema 尽量统一，具体能力差异由客户端实现和 service 层处理

### “当前表结构”不等于“唯一数据来源”

- 某些表承担缓存或兼容职责，例如 `yande_images`、`booru_tags.updatedAt`
- 某些字段是为跨页面 UI 状态持久化服务，例如 `booru_posts.isLiked`
