# 架构总览

## 项目定位

Yande Gallery Desktop 是一个基于 Electron + React + TypeScript 的桌面应用，核心目标是同时服务两类场景：

1. 本地图库管理：扫描、检索、分组、缩略图、预览与标签管理。
2. Booru 浏览与下载：统一接入 Moebooru、Danbooru、Gelbooru 等站点，支持搜索、收藏、下载、缓存和批量任务。

## 整体分层

### 主进程 `src/main/`

- `services/`：数据库、配置、图库扫描、缩略图、下载、缓存、Booru 客户端、Google 相关服务。
- `ipc/`：对外暴露统一 IPC 通道，承接渲染进程请求。
- `window.ts`：窗口创建和运行时配置。

### 渲染进程 `src/renderer/`

- `App.tsx`：主导航、固定页面、主题、快捷键、页面切换。
- `pages/`：图库、Booru、Google、设置等页面。
- `components/`：网格、详情区块、表单、浮层、排序菜单等复用组件。
- `utils/`：查询拼装、多选逻辑、标签关系、查看器控制等纯函数。

### 预加载层 `src/preload/`

- 使用 `contextBridge.exposeInMainWorld` 暴露受控 API。
- 统一承接 `db.*`、`gallery.*`、`config.*`、`image.*`、`booru.*`、`system.*` 等接口。

### 共享层 `src/shared/`

- 保存本地图库、Booru、批量下载、标签体系等共享类型定义。

## 当前模块版图

### 本地图库

- `imageService.ts`：图片导入、搜索、标签关联。
- `galleryService.ts`：图库 CRUD、封面、按目录扫描。
- `thumbnailService.ts`：缩略图生成与缓存。

### Booru 子系统

- `booruClientInterface.ts`：统一客户端接口。
- `moebooruClient.ts`、`danbooruClient.ts`、`gelbooruClient.ts`：站点适配层。
- `booruClientFactory.ts`：按站点类型分发客户端。
- `booruService.ts`：本地持久化、收藏、标签、保存的搜索、分组、黑名单等。
- `downloadManager.ts`、`bulkDownloadService.ts`：单任务与批量任务下载调度。
- `imageCacheService.ts`：Booru 图片缓存与统计。
- `backupService.ts`：应用数据导出与恢复。
- `imageMetadataService.ts`：查看器元数据读取。

### 辅助集成

- `googleAuthService.ts`、`googleDriveService.ts`、`googlePhotosService.ts`：Google 账号与资源接入。
- 渲染层包含 `GoogleDrivePage.tsx`、`GooglePhotosPage.tsx`、`GoogleAccountPage.tsx`、`GeminiPage.tsx` 等页面入口。

## 数据与存储

### 路径体系

- 配置根目录由 `.env` 中的 `CONFIG_DIR` 决定；未指定时默认落到用户目录下的 `.yandegallery`。
- `config.yaml` 位于配置根目录。
- `dataPath` 决定运行数据目录，数据库、缩略图、缓存、日志围绕它解析。

相关说明见：

- `doc/开发与配置指南.md`
- `doc/注意事项/配置路径解析系统.md`
- `doc/注意事项/数据迁移规则.md`

### 数据库

- SQLite 是主数据存储。
- `src/main/services/database.ts` 是当前 schema 的实际来源。
- 本地图库、Booru 记录、批量下载、收藏标签、黑名单、保存的搜索等都持久化在同一个数据库中。

详情见 `doc/数据库结构文档.md`。

## 关键设计原则

### 1. 外部网络请求走主进程

渲染进程不直接请求外部站点，统一通过 IPC 交给主进程执行，以规避 CORS、集中处理代理和错误。

### 2. 组件抽象优先于页面复制

图片网格、列表包装、Booru 详情区块、批量下载卡片等能力尽量以可复用组件沉淀。

### 3. 本地图库和 Booru 记录并行存在

本地 `images` 与 Booru `booru_posts` 分开建模，但通过 `localImageId` 等字段建立关联，兼顾本地管理与远端元数据。

### 4. 配置与运行数据分离

路径解析不再依赖编译目录，减少打包后路径失效和不可写问题。
