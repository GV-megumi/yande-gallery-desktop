# 开发与配置指南

## 环境要求

- Node.js 18+
- npm 9+
- Git
- Python 3（用于原生模块编译）
- Windows 下建议安装 Visual Studio Build Tools 或 Visual Studio Community 的 C++ 构建组件

## 安装依赖

```bash
npm install
```

如果原生模块编译有问题，优先看 `doc/注意事项/Native模块编译问题.md`。

## 常用命令

```bash
# 开发
npm run dev

# 构建
npm run build

# 测试
npm run test

# 原生模块重建
npm run rebuild
npm run rebuild:sharp

# 打包
npm run dist
npm run dist:win
npm run dist:mac
npm run dist:linux
```

## 配置文件与路径

### 路径解析规则

1. 先读取 `.env` 中的 `CONFIG_DIR`。
2. `config.yaml` 位于 `CONFIG_DIR` 下。
3. `config.yaml` 中的 `dataPath` 决定运行数据目录。
4. 数据库、缩略图、缓存、日志再分别基于配置目录或数据目录解析。

### 默认结构

```text
CONFIG_DIR/
  config.yaml
  gallery.db
  data/
    thumbnails/
    cache/
    app.log
```

### 关键配置项

```yaml
dataPath: data

database:
  path: gallery.db
  logging: true

downloads:
  path: downloads
  createSubfolders: true

thumbnails:
  cachePath: thumbnails
  maxWidth: 800
  maxHeight: 800
  quality: 92
  format: webp

network:
  proxy:
    enabled: false
    protocol: http
    host: 127.0.0.1
    port: 7890
```

详细规则见：

- `doc/注意事项/配置路径解析系统.md`
- `doc/注意事项/数据迁移规则.md`

## 开发调试

### 主进程

- 主进程日志输出在终端。
- 关键服务都带模块前缀日志。

### 渲染进程

- Electron 窗口内按 `F12` 打开开发者工具。

### 测试建议

- 文档变更不要求构建，但如果同步修改了配置示例或命令说明，建议至少确认 `package.json`、实际路径和脚本名称一致。

## 常见问题

### 1. 原生模块安装失败

- `sqlite3` / `sharp` 与 Electron 版本绑定，升级 Electron 后通常需要重建。

### 2. 代理不生效

- 检查 `config.yaml` 中 `network.proxy` 配置。
- 确认代理服务已启动。
- 确认网络请求是经由主进程发起，而不是渲染进程直接请求。

### 3. 文档里提到的路径和实际目录不一致

- 以 `src/main/services/config.ts` 和 `src/main/services/database.ts` 的实际实现为准。
