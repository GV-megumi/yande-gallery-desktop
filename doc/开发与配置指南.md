# 开发与配置指南

## 环境要求

- Node.js 18+
- npm 9+
- Git
- Python 3（用于原生模块编译）
- Windows 下建议安装 Visual Studio Build Tools 或 Visual Studio Community 的 C++ 构建组件

## 安装依赖

```bash
npm install
```

如果原生模块编译有问题，优先看 `doc/注意事项/Native模块编译问题.md`。

## 常用命令

```bash
# 开发
npm run dev

# 构建
npm run build

# 测试
npm run test

# 原生模块重建
npm run rebuild
npm run rebuild:sharp

# 打包
npm run dist
npm run dist:win
npm run dist:mac
npm run dist:linux
```

## 配置文件与路径

### 路径解析规则

1. 先读取 `.env` 中的 `CONFIG_DIR`。
2. `config.yaml` 位于 `CONFIG_DIR` 下。
3. `config.yaml` 中的 `dataPath` 决定运行数据目录。
4. 数据库、缩略图、缓存、日志再分别基于配置目录或数据目录解析。

### 默认结构

```text
CONFIG_DIR/
  config.yaml
  gallery.db
  data/
    thumbnails/
    cache/
    app.log
```

注意：`gallery.db` 默认不在 `data/` 目录里，而是和 `config.yaml` 同级；`thumbnails`、`cache`、日志这类运行数据默认放在 `data/` 下。

### 关键配置项

```yaml
dataPath: data

database:
  path: gallery.db
  logging: true

downloads:
  path: downloads
  createSubfolders: true

thumbnails:
  cachePath: thumbnails
  maxWidth: 800
  maxHeight: 800
  quality: 92
  format: webp

network:
  proxy:
    enabled: false
    protocol: http
    host: 127.0.0.1
    port: 7890
```

除了上面这些，当前配置还实际包含：

- `ui.menuOrder`：菜单排序持久化
- `ui.pinnedItems`：固定页面持久化
- `booru.appearance`：网格大小、预览质量、分页位置、页面模式、缓存上限等
- `booru.download.tokenDefaults`：文件名模板 token 默认选项

详细规则见：

- `doc/注意事项/配置路径解析系统.md`
- `doc/注意事项/数据迁移规则.md`

## 开发调试

### 主进程

- 主进程日志输出在终端。
- 关键服务都带模块前缀日志。
- 应用启动时会自动执行：路径初始化、配置加载、数据库初始化、按配置创建初始图库、后台恢复普通下载和批量下载、清理过期标签缓存。

### 渲染进程

- Electron 窗口内按 `F12` 打开开发者工具。

### 测试建议

- 文档变更不要求构建，但如果同步修改了配置示例或命令说明，建议至少确认 `package.json`、实际路径和脚本名称一致。

## 启动时会自动做什么

应用启动后，主进程会自动执行这些动作：

1. 读取 `.env` 并初始化 `CONFIG_DIR`
2. 加载 `config.yaml`
3. 确保配置目录和数据目录存在
4. 初始化数据库并执行兼容迁移
5. 如果数据库里还没有图库记录，则按配置创建初始图库
6. 后台恢复普通下载和批量下载会话
7. 清理过期标签缓存

这也是为什么应用重启后部分下载任务会自动恢复，不需要手动重新点开始。

## Renderer API / 预加载能力

当前渲染层并不是直接访问 Node 或主进程，而是通过 `window.electronAPI` 访问这些域：

- `db`
- `gallery`
- `config`
- `image`
- `booru`
- `bulkDownload`
- `window`
- `system`

如果需要补页面能力文档，`src/preload/index.ts` 是最直接的入口清单。

另外，`window.electronAPI` 里还有几类事件订阅接口，而不只是普通调用：

- 配置变更监听：`config.onConfigChanged`（主进程只广播 `ConfigChangedSummary` 摘要 `{ version, sections }`；preload 收到摘要后会自动重新拉取去敏配置并回调，事件通道不再下发完整 config）
- Booru 外观偏好变更监听：`booruPreferences.appearance.onChanged`（同样基于上述摘要事件，preload 自动拉取最新 appearance DTO）
- 收藏修复完成：`booru.onFavoritesRepairDone`
- 下载事件：`booru.onDownloadProgress`、`booru.onDownloadStatus`、`booru.onQueueStatus`
- 批量下载事件：`system.onBulkDownloadRecordProgress`、`system.onBulkDownloadRecordStatus`

## IPC 入口说明

- `src/main/ipc/channels.ts` 是主要的 IPC 常量清单，但当前并不是唯一来源。
- 一部分通道名仍直接写在 `src/preload/index.ts` 中。
- 所以在新增或核对 IPC 能力时，建议同时查看 `src/preload/index.ts`、`src/main/ipc/channels.ts` 和对应 handler 文件。

## 主进程启动补充

除了 `initializeApp()` 内部流程，主进程启动还包括：

- 注册 `app://` 自定义协议
- 单实例锁处理
- 禁用硬件加速
- 创建窗口与注册 IPC

## 常见问题

### 1. 原生模块安装失败

- `sqlite3` / `sharp` 与 Electron 版本绑定，升级 Electron 后通常需要重建。

### 2. 代理不生效

- 检查 `config.yaml` 中 `network.proxy` 配置。
- 确认代理服务已启动。
- 确认网络请求是经由主进程发起，而不是渲染进程直接请求。

### 3. 文档里提到的路径和实际目录不一致

- 以 `src/main/services/config.ts` 和 `src/main/services/database.ts` 的实际实现为准。
