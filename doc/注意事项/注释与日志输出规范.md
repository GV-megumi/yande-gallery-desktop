# 注释与日志输出规范

## 注释

### 适合写注释的地方

- 文件职责。
- 复杂函数的用途、参数和返回值。
- 不明显的业务分支或兼容逻辑。

### 不适合写注释的地方

- 明显到代码本身已经表达清楚的语句。
- 会和代码一起频繁漂移、容易过期的逐行复述。

## 日志

### 推荐格式

```ts
console.log('[模块名] 操作描述:', data);
console.warn('[模块名] 警告信息:', data);
console.error('[模块名] 操作失败:', error);
```

### 建议打日志的动作

- 数据加载开始、成功、失败。
- 网络请求发起和异常。
- 文件读写、扫描、下载、缓存处理。
- 用户触发的重要操作。

## 目标

- 出问题时能快速定位模块。
- 保持日志粒度足够排查，但不要把简单渲染噪音全打出来。
