# 配置路径解析系统

## 核心原则

路径解析不依赖编译输出目录，也不默认依赖项目根目录，而是围绕配置根目录和数据目录两层展开。

## 解析链路

1. `.env` 的 `CONFIG_DIR` 决定配置根目录。
2. `config.yaml` 从配置根目录读取。
3. `config.yaml` 的 `dataPath` 决定运行数据目录。
4. 数据库、缩略图、缓存、日志再分别解析。

## 两类路径函数

- `resolveConfigPath()`：适合数据库等与 `config.yaml` 同级的文件。
- `resolveDataPath()`：适合缩略图、缓存、日志等运行数据。

## 这样设计的原因

- 避免 `__dirname` 在打包后进入只读或不可预期位置。
- 避免 `process.cwd()` 在不同启动方式下不稳定。
- 允许用户把配置和运行数据迁移到自定义目录。
