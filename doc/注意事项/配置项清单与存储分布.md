# 配置项清单与存储分布

> 目的：盘点当前应用**所有"配置性"信息**，弄清每一项「存在哪里、由谁写入、有没有界面、**代码到底有没有读取它**」，
> 为后续"哪些配置该清理 / 该补界面 / 该挪进数据库"的取舍提供事实依据。
>
> 本文经过一次**逐字段源码审计**（grep 每个字段的真实读取方），与 §三 的"死配置审计"是本文最重要的产出。
>
> Source of truth：`src/main/services/config.ts`（schema 与默认值）、`src/main/services/database.ts`（库表）、
> `src/renderer/pages/SettingsPage.tsx` 与 `src/renderer/pages/BooruSettingsPage.tsx`（设置界面）。
> 本文如与代码冲突，以代码为准。

> ✅ **更新（2026-06）**：§三 列出的死配置已全部从 `config.ts`、`backupService.ts`、`config.example.yaml`
> 及相关测试中**删除**。下面 §二 表格为清理后的现状；§三 保留为审计与历史记录。

## 一、存储介质总览

配置散布在 **6 种介质**里：

| # | 介质 | 位置 | 管理者 | 形态 | 典型内容 |
|---|------|------|--------|------|---------|
| 1 | `config.yaml` | `CONFIG_DIR/config.yaml` | 主进程 `config.ts` | 全局**单例**、强类型 | 路径、缩略图、代理、通知、桌面行为、Booru 外观/命名、API 服务 |
| 2 | SQLite `gallery.db` | `CONFIG_DIR/gallery.db` | `database.ts` + 各 service | **多条目**业务表 | Booru 站点与凭据、收藏/黑名单标签、saved search、下载绑定、批量任务、本地图库 |
| 3 | 渲染层 `localStorage` | 浏览器存储（每用户） | 渲染层 hook | 轻量 KV | 主题、语言 |
| 4 | 硬编码常量 | 源码内 `const` | —— | 改代码才能调 | 超时、并发、分页、缓存上限、UI design token、窗口尺寸 |
| 5 | 环境变量 / `.env` | `CWD/.env`、系统环境 | `config.ts` 等 | 启动期读取 | `CONFIG_DIR`、`NODE_ENV`、`HTTP(S)_PROXY`、devtools 开关 |
| 6 | 操作系统 | 登录项、系统代理 | `app.setLoginItemSettings` / 注册表 | 间接同步 | 开机自启（随 `desktop.autoLaunch`）、系统代理回退 |

**关键事实：项目没有通用的 KV 设置表**（无 `settings`/`preferences`/`meta` 表），库里的配置都落在各自的业务表上。

## 四、🟢 全局设置界面（`SettingsPage.tsx`）

齿轮设置页用 Segmented 分 **4 个一级 Tab**（`SettingsPage.tsx:647-656`）：

### Tab 1 · 通用配置
| 分组 | 设置项 | 绑定 / 持久化 |
|------|--------|--------------|
| 图库文件夹 | 增删本地图库（创建/彻底删除/停止监视）、扫描子文件夹、忽略名单 | → DB `galleries` 表（`gallery.createGallery` / `gallery.deleteGallery` / `gallery.updateGallery`）；忽略名单 → DB `gallery_ignored_folders` |
| 下载设置 | 下载路径 | `downloads.path` |
| 缩略图设置 | 尺寸 / 质量 / 压缩强度 | `thumbnails.maxWidth+maxHeight` / `quality` / `effort` |
| 外观 | **主题 / 语言** | ⚠ 走 `localStorage`，**不进 config.yaml** |
| 通知 | 总开关 + 分类 + 单张 + 点击行为 | `config:set-notifications` → `notifications.*` |
| 桌面行为 | 关闭行为 / 开机自启 / 启动最小化 / 硬件加速 | `config:set-desktop` → `desktop.*`（自启另调 OS 登录项）|
| 备份与恢复 | 导出 / 合并导入 / 替换导入 | `system:export-backup` / `system:import-backup`（操作的是 **DB** 数据）|

### Tab 2 · 代理配置
- 代理服务器：启用 / 协议 / 主机 / 端口 → `network.proxy.*`（账号密码无界面，但生效）
- 连通性测试：测百度 / 测 Google（`system:test-baidu` / `system:test-google`，纯测试不落配置）

### Tab 3 · API 服务
- 启用 / 监听模式 / 端口 → `apiService.*`
- API Key：生成 / 显隐 / 复制（保存走 `api-service:save-config` → config.yaml）
- 权限：11 个开关 → `apiService.permissions.*`
- 日志：启用 / 界面可见 → `apiService.logs.*`（日志本体进 DB `api_logs`）

### Tab 4 · 关于
应用信息、版本、检查更新、GitHub 链接（无配置写入）。

> 注意：设置页里出现的 **Google 仅是"测试 Google 连通性"按钮**，与 §三 的死配置 `google` 块无关，不要混淆。

## 五、🔵 Booru 设置界面（`BooruSettingsPage.tsx`）

| 设置项 | 字段 | 备注 |

## 七、隐式配置（无界面 / 写死 / 自动写回）

用户问的"隐式配置"主要是这几类。注意区分「⚪ 无界面但生效」与「❌ 死配置」（后者见 §三）。

### 7.6 硬编码"魔法常量"（想调但目前改不了）
数量很多（40+），最值得关注的几类：

- **网络/下载**：单文件下载超时 `600000ms`、无数据超时 `120000ms`、页面请求超时 `10000ms`、重试 `maxRetries=3`/`maxDownloadRetries=2`、浏览模式并发 `BROWSING_MODE_CONCURRENCY=1`（`downloadManager.ts` / `bulkDownloadService.ts`）

- **缓存**：缓存并发 `MAX_CACHE_CONCURRENCY=8`、单文件上限 `MAX_SINGLE_FILE_SIZE_MB=200`、目录大小检查间隔 `300000ms`（`imageCacheService.ts`）
- **每页上限**：Gelbooru `100` / Danbooru `200` / Moebooru `1000`（`bulkDownloadService.ts:1430`）
- **前端渲染**：最近图初始加载 `2000`、可见批 `200`、缩略图并发 `4`、搜索防抖 `300ms`、会话轮询 `5000ms`（`GalleryPage.tsx` / `ImageGrid.tsx` / `useDebounce.ts`）
- **窗口**：主窗最小 `1200×700`、子窗上限 `MAX_SUB_WINDOWS=10`（`window.ts`）
- **UI design token**：spacing/layout/radius/fontSize/zIndex（`src/renderer/styles/tokens.ts`）—— 主题级常量，一般不暴露给用户。
