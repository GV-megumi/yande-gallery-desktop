# 注意事项索引

这个目录集中放“为什么这样设计、改代码时要注意什么、哪些地方容易踩坑”这类说明，避免它们继续散落在功能文档和历史文档里。

- `网络访问与CORS解决方案.md`：为什么外部请求统一走主进程，以及 IPC 代理模式的边界
- `配置路径解析系统.md`：`CONFIG_DIR`、`config.yaml`、`dataPath` 的路径解析规则
- `数据迁移规则.md`：旧路径、旧数据目录、旧配置格式如何兼容迁移
- `代码修改规范.md`：改逻辑前如何评估影响范围、依赖和兼容性
- `组件抽象与复用原则.md`：哪些 UI 能力应沉淀成复用组件，为什么要这样做
- `Moebooru开发规范.md`：开发 Moebooru 能力时为什么要先参考 Boorusama
- `图库懒加载设计.md`：图库为什么采用元数据先行、按需扫描的设计
- `API限流与安全约束.md`：多站点请求、认证、代理和安全边界的基本约束
- `代理配置指南.md`：代理配置入口、适用场景和排查方向
- `Native模块编译问题.md`：`sqlite3`、`sharp` 等原生模块常见安装/重建问题
- `注释与日志输出规范.md`：什么地方该写注释、日志建议怎么打
- `性能约束与反模式.md`：性能审计报告的摘要入口，方便开发前快速自查

如果你先从总览文档进入，推荐按这个顺序继续读：

1. `doc/架构总览.md`
2. `doc/开发与配置指南.md`
3. 再按专题进入这里的单篇说明
