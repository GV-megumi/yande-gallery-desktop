# 组件抽象与复用原则

## 核心原则

必要的 UI 能力要沉淀成可复用组件或容器，而不是在页面里复制粘贴实现。

## 已经体现出的模式

- `ImageGrid`
- `ImageListWrapper`
- `ImageSearchBar`
- `LazyLoadFooter`
- `BooruImageCard`
- `BooruPostDetails/*`
- `BulkDownload*`

## 为什么重要

- 修改一处可以覆盖多个使用场景。
- 页面层更聚焦状态和流程，组件层更聚焦渲染和交互。
- 降低不同页面之间行为和样式不一致的问题。

## 适用判断

- 同一能力在两个以上页面重复出现。
- UI 和交互规则稳定，值得沉淀。
- 需要统一 loading/empty/error/list 这类通用状态处理。
