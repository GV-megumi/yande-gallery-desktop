# 代码修改规范

## 修改前必须做的事

1. 先确认影响范围。
2. 查找调用方或依赖方。
3. 判断是否需要保持向后兼容。

## 重点原则

- 不要只改表面调用，不看底层实现。
- 修改公共接口时，优先用可选参数或默认值维持旧行为。
- 有底层改动时，要同步核对 IPC、Preload、页面层和类型定义。

## 推荐流程

1. 读定义。
2. 查引用。
3. 列出受影响文件。
4. 逐步修改。
5. 回头验证相关功能没有被带坏。

## 典型风险

- 改了 service 返回结构，但 preload / renderer 没同步。
- 改了数据库字段，但类型和查询文档没同步。
- 修一个 bug 时顺手重构太多，导致回归范围失控。
