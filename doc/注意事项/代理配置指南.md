# 代理配置指南

## 适用场景

- 当前网络无法直接访问 Google 或部分 Booru 站点。
- 需要统一代理主进程发出的 HTTP 请求。

## 生效优先级

1. 设置页启用的自定义代理，即 `config.yaml` 中 `network.proxy.enabled: true`。
2. 系统默认代理。Windows 下读取当前用户的 Internet Settings 代理开关、代理地址和绕过列表；其他平台读取常见代理环境变量。
3. 如果自定义代理未启用，且系统代理未开启或没有可用地址，则直连。

## 系统代理兼容项

- Windows：读取 `ProxyEnable`、`ProxyServer`、`ProxyOverride`，支持分协议代理、`<local>`、通配符和端口限定的绕过规则。
- 其他平台：读取 `HTTP_PROXY`、`HTTPS_PROXY`、`ALL_PROXY`，并遵守 `NO_PROXY` / `no_proxy`。

## 自定义代理配置

```yaml
network:
  proxy:
    enabled: true
    protocol: http
    host: 127.0.0.1
    port: 7890
```

## 原则

- 代理配置应由主进程读取并应用。
- 设置页关闭“自定义代理”不代表强制直连，而是回退到系统默认代理。
- 代理问题先检查自定义代理配置，再检查系统代理开关和代理服务是否运行。

## 不建议的做法

- 在页面层自行拼接代理 URL。
- 让不同服务各自维护不同代理配置入口。
