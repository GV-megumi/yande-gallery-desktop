# 功能总览

## 当前功能范围

本项目现在已经不只是早期的 Yande.re 浏览器，而是一个覆盖本地图库、Booru 多站点、批量下载、辅助云接入的桌面应用。

## 本地图库

### 已有能力

- 最近图片：懒加载浏览。
- 全部图片：分页浏览、按文件名或标签搜索。
- 图集管理：按目录组织，支持封面设置。
- 图集文件夹同步：对已有图集重新扫描目录，自动导入新增文件并更新统计。
- 图集删除与忽略名单（v0.0.2 起）：删除图集会按 `recursive` 字段级联清理关联 DB / 缩略图；同时把目录加入 `gallery_ignored_folders`，下次扫描不会重建。原始图片文件不会被物理删除。
- 无效图片管理：标记、查看、清理无效图片记录。
- 标签系统：标签增删、搜索、批量关联。
- 缩略图系统：WebP 缩略图缓存。
- 预览能力：缩放、拖拽、键盘导航。
- 图集卡片右键"用单独窗口打开"（v0.0.2 起）：通过 `window.openSecondaryMenu` 在新子窗口直接打开某个图集详情，附加上下文走 `extra` query。

### 相关模块

- 页面：`src/renderer/pages/GalleryPage.tsx`
- 服务：`src/main/services/imageService.ts`、`src/main/services/galleryService.ts`、`src/main/services/thumbnailService.ts`

## Booru 多站点

### 站点适配

- Moebooru
- Danbooru
- Gelbooru

### 基础浏览与搜索

- 帖子列表、单帖详情、分页与搜索。
- 热门帖子、Pools、标签详情页。
- 标签自动补全、高级搜索语法、高级过滤器。
- 随机帖子、相关标签推荐。

### 深化浏览能力

- 艺术家页、角色页、Wiki 页面、论坛浏览、用户主页。
- 帖子注释覆盖层、帖子版本历史。
- 视频帖子支持。
- 高级图片查看器：旋转、翻转、对比模式、元数据查看。

### 收藏与组织

- 本地收藏。
- 服务端收藏。
- 收藏夹分组。
- 收藏标签、标签分组。
- 黑名单标签。
- 保存的搜索。
- 服务端收藏、投票、评论、举报（按站点能力区分）。

### 下载与缓存

- 单帖下载。下载完成前若站点返回了 `md5`，会在临时文件替换目标文件前做一次 md5 校验（v0.0.2 起，TP-03）。
- 下载队列：暂停、恢复、重试、状态查看。
- 批量下载任务与会话管理。
- 批量下载并发闸门与等待队列（v0.0.2 起）：同一时刻活跃会话超过 `bulkDownload.maxConcurrentSessions` 时，新 `startSession` 会写入 `queued` 状态；任何会话离开活跃态后会尝试 `promoteNextQueued` 推进队首，进程重启恢复逻辑也走同一闸门。
- 收藏标签一键下载：为收藏标签配置下载绑定，一键创建批量下载任务；相同路径 + 标签的任务自动去重。
- 缓存统计与清理。
- 文件名模板系统。
- 数据备份与恢复。

### 其他高级能力

- 标签关系（alias / implication）浏览。
- 图片元数据读取。
- 论坛主题与帖子浏览。
- 帖子注释（notes）和版本历史。

## Google / 应用扩展区

- Google Drive 页面：通过 webview 嵌入 drive.google.com。
- Google Photos 页面：通过 webview 嵌入 photos.google.com。
- Gemini 页面：通过 webview 嵌入 gemini.google.com。

这部分能力与 Booru、本地图库并列挂在应用主导航中，属于辅助集成功能。所有 Google 页面均采用 webview 嵌入方式，不依赖 Google API 或 OAuth 认证。

## 页面合并与页内子导航

部分 Booru 二级菜单已合并为 Hub 页面，通过 Ant Design `Segmented` 组件实现页内 tab 切换，减少侧边栏条目数量：

- **下载中心**（`BooruDownloadHubPage`）：合并"下载管理"和"批量下载"
- **标签管理**（`BooruTagManagementPage`）：合并"收藏标签"和"黑名单"

Hub 页面同时挂载两个子页面（用 `display:none` 隐藏非活跃页），切换时不丢失各自状态。

## 桌面行为与通知（v0.0.2 起）

- **系统托盘**：主窗口支持缩到托盘，托盘菜单提供"显示主窗口 / 退出"等入口。
- **关闭行为**：`desktop.closeAction` 控制点主窗口 X 时是 `hide-to-tray`、`quit` 还是 `ask`（弹确认）。
- **开机自启**：`desktop.autoLaunch` 开启后调用 `app.setLoginItemSettings`；`desktop.startMinimized` 控制自启时是否直接隐藏到托盘。
- **桌面通知**：由 `notifications` 三级开关控制——全局 `enabled`、按会话终态 `byStatus.{completed,failed,allSkipped}`、任务级 `notifications` 字段；`singleDownload.enabled` 单独决定单图下载是否弹通知；`clickAction` 控制点通知后跳转到哪里（`focus` / `openDownloadHub` / `openSessionDetail`）。
- **异常退出治理**：主进程接入 `before-quit` / `will-quit` / `render-process-gone` / `uncaughtException` / `unhandledRejection`，确保托盘图标、子窗口、托管定时器在退出前被正确清理。

对应设置项见"设置页 → 通知 / 桌面行为"两个 Tab；能力闭环约束见 `doc/注意事项/下载与批量会话状态机.md` 第 6 节。

## 子窗口

任意二级菜单页面均可通过 `window.openSecondaryMenu(section, key, tab?, extra?)` 在独立子窗口中打开，由 `SubWindowApp.tsx` 负责路由和渲染。`extra` 用来把必要上下文（例如 `{ galleryId }`）显式带进子窗口，避开共享 `pagePreferences` 带来的污染；保留键 `section` / `key` / `tab` 不会被透传。子窗口使用 `React.lazy` 按需加载页面组件，不包含侧边栏。

轻量子窗口（`tag-search` / `artist` / `character`）加载的是精简 preload，只暴露 `window` / `booru` / `booruPreferences` / `system` 四个域；二级菜单子窗口仍使用主 preload。详见 `doc/Renderer API 文档.md` 的"主 / 子窗口 preload 暴露面差异"。

## 设置 / 关于

- 设置页的"关于" Tab 有"检查更新"入口，拉取 GitHub Releases API 的最新版本并对比（v0.0.2 起）。成功结果在主进程做约 60s 的短时缓存，错误不缓存以便重试。
