- `BlacklistedTagsPage` 的页面偏好恢复与保存链路，已改为经由 `window.electronAPI.pagePreferences.blacklistedTags.get/save` 执行，不再从页面内直接读写整包 `config.get/save`
- `GalleryPage` 的页面偏好恢复与保存链路，已改为经由 `window.electronAPI.pagePreferences.gallery.get/save` 执行，不再从页面内直接读写整包 `config.get/save`
- `App.tsx` 的菜单排序与固定项恢复/保存链路，已改为经由 `window.electronAPI.pagePreferences.appShell.get/save` 执行，不再从 App 壳层直接读写整包 `config.get/save`
- preload 已暴露专用 `pagePreferences.favoriteTags` / `pagePreferences.blacklistedTags` / `pagePreferences.gallery` / `pagePreferences.appShell` bridge，主进程已补齐对应正式 IPC 通道与 handler
- `FavoriteTagsPage` 激活与重新激活时，会先经由专用偏好接口 hydrate，再写回新的筛选 / 排序 / 分页状态
- `BlacklistedTagsPage` 激活与重新激活时，会先经由专用偏好接口 hydrate，再写回新的筛选 / 关键字 / 分页状态
- `GalleryPage` 在 `all` / `galleries` 两个高频子页中，会先经由专用偏好接口恢复搜索 / 分页 / 排序 / `selectedGalleryId` 等工作上下文；hydrate 后不会立即回写相同偏好，用户修改后才保存新偏好
- `App.tsx` 启动时会从 `pagePreferences.appShell` 恢复主菜单 / 二级菜单顺序与固定项，并在 booru 旧 key 迁移后回写新偏好源，而不是继续依赖旧 `ui.menuOrder` / `ui.pinnedItems`
- App Shell booru 历史固定项迁移已补齐“迁移后去重并继续向后补满最多 5 个唯一项”的回归，避免先截断再丢失后续唯一项
- 页面偏好保存成功后，会继续广播 `CONFIG_CHANGED`；广播载荷保持为 safe config，而不是把敏感字段或 App Shell UI 状态直接送到 renderer
- `PAGE_PREFERENCES_SAVE_FAVORITE_TAGS`、`PAGE_PREFERENCES_SAVE_BLACKLISTED_TAGS`、`PAGE_PREFERENCES_SAVE_GALLERY` 与 `PAGE_PREFERENCES_SAVE_APP_SHELL` 的自动化断言已覆盖：成功时只写入对应页面偏好，且 App Shell 变更不再回退到整包 `config.save`
- `src/main/services/config.ts` 已补齐 `pagePreferences.appShell.pinnedItems` 的主进程/配置层统一收口：`normalizeConfigSaveInput()` 与 `rebuildPagePreferences()` 现在都会把固定项数量限制为最多 5 个，因此即使绕过 renderer 直接写入 `PAGE_PREFERENCES_SAVE_APP_SHELL`，也不会把超限固定项写入配置

   - FavoriteTags 首切片：6 个测试文件、73 个测试通过；
   - BlacklistedTags 切片回归组合：4 个测试文件、50 个测试通过；

3. 本轮可收口固化的结论包括：
   - FavoriteTags 激活时会通过专用 `pagePreferences.favoriteTags` 接口恢复筛选 / 排序 / 分页状态；
   - BlacklistedTags 激活时会通过专用 `pagePreferences.blacklistedTags` 接口恢复筛选 / 关键字 / 分页状态；
   - Gallery 在 `all` / `galleries` 两个高频子页中会通过专用 `pagePreferences.gallery` 接口恢复搜索 / 分页 / 排序 / `selectedGalleryId` 等工作上下文；
