# 重构文档

这套文档不是 `审查报告.md` 的拆页副本，而是把已有审查结果重组为一套**可执行、可分发、可复核**的重构说明体系。

## 目标

- 为后续重构提供统一入口与导航
- 支持按阶段推进，而不是按原报告章节机械执行
- 支持将任务分发给不同子智能体并行实施
- 支持人工逐项推进与阶段性验收
- 支持后续独立复核智能体再次审查并整合结论

## 使用原则

1. **`审查报告.md` 仍是 source of truth**
   - 新文档体系负责重组和执行，不负责篡改原报告结论。

2. **一个问题只能有一个主责任文档**
   - 其他文档只做引用，不重复大段展开。

3. **主组织轴是实施阶段 / 依赖关系**
   - P0/P1/P2 继续保留，但不再作为唯一组织方式。

4. **先修状态与边界，再修体验与增强**
   - 下载状态机、协议边界、安全硬伤、启动/退出治理优先于 UI 美化或长期增强。

5. **专项文档用于直接执行，总控文档用于排序和协调**
   - 想开工，优先看 `03 + 对应专项文档`
   - 想判断先后顺序，优先看 `02`
   - 想判断是否漏项，优先看 `04`

## 与原报告的关系

- 原报告偏“审查视角”，强调发现问题、给出证据、划分级别
- 本套文档偏“执行视角”，强调如何按阶段推进、如何拆任务、如何验收、如何复核
- 两者不是替代关系，而是“原报告给结论，这套文档给落地路径”
